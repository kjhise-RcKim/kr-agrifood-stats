# 농식품 주요통계 대시보드 — 핵심지표 구조 초안 (v0.1)

> 원자료: 농림축산식품부 『농림축산식품 주요통계 2025』 (581p, 공공누리 출처표시)
> 이 문서는 데이터 구조·핵심지표를 정의하는 설계 초안입니다. 실제 수치는 아직 담지 않았습니다.

---

## 0. 설계 원칙 (가장 중요)

**대시보드·검색·AI 분석이 전부 "같은 JSON 하나"를 읽게 만든다.**
- 지표 하나 = 레코드 하나. 대시보드는 `series`를 그리고, 검색은 `name`·`keywords`를 뒤지고, AI는 전체 필드 + `related_ids`로 지표를 엮고 `source`로 출처를 인용한다.
- 원자료가 공공누리 **출처표시** 조건이므로, 모든 레코드에 `source`(장·페이지)를 반드시 넣는다 → AI 조언이 근거를 인용할 때 그대로 사용.

## 2. 대시보드 카테고리 (10개 그룹)

일반 공개용이므로, 사람들이 직관적으로 찾는 순서로 재편성했습니다. (원문 편제와 1:1이 아니라 "사용자 관점"으로 묶음)

1. **농업 총괄** — 생산액, 농가·인구, 경지, GDP 비중, 농가소득
2. **식량·자급률** — 식량/곡물 자급률, 쌀·주요 식량작물
3. **물가·가격** — 농축산물 물가, 식료품 소비자물가, 농가 교역조건
4. **축산** — 사육두수, 축산물 생산·소비·자급률
5. **채소·과일(경제작물)** — 주요 품목 생산·면적
6. **무역** — 수출·수입·무역수지, 주요 품목
7. **임업** — 임산물 생산, 목재 자급률
8. **어업** — 생산량, 어가소득
9. **식품산업** — 산업 규모, 외식·가공
10. **농촌·사람** — 고령화율, 귀농·귀촌
11. **국제비교** — 주요국(일본·대만·중국) 농업 비교 *(8절)*
12. **남북한비교** — 북한 농업·인구·무역 *(8절)*

## 4. 데이터 스키마 (JSON)

### 4-1. 지표 레코드 (indicator)

```json
{
  "id": "farm_income",
  "name": "농가소득(호당)",
  "name_en": "Farm household income",
  "group": "농업 총괄",
  "unit": "천원",
  "frequency": "연간",
  "is_headline": true,
  "description": "농가 1호당 연간 소득(농업소득+농외소득+이전소득+비경상소득)",
  "keywords": ["농가소득", "소득", "농가경제", "농업소득", "농외소득"],
  "source": {
    "publication": "농림축산식품 주요통계 2025",
    "section": "기본통계 > Ⅶ. 농가경제",
    "page": 337,
    "org": "통계청 농가경제조사",
    "license": "공공누리 출처표시"
  },
  "series": [
    { "year": 2022, "value": 46153 },
    { "year": 2023, "value": 47214 },
    { "year": 2024, "value": null, "flag": "p" }
  ],
  "breakdown": null,
  "related_ids": ["agri_output_value", "farm_population", "farmer_aging_rate"],
  "notes": "단위 미만은 0, 잠정치는 flag='p'"
}
```

### 4-2. 하위분류가 있는 경우 (`breakdown` 사용)

가축 사육두수처럼 품목이 여러 개면 `series` 대신 `breakdown` 안에 품목별 시계열을 넣습니다.

```json
{
  "id": "livestock_heads",
  "name": "주요 가축 사육두수",
  "unit": "천두",
  "breakdown": [
    { "label": "한·육우", "series": [ {"year":2023,"value":3554}, {"year":2024,"value":null,"flag":"p"} ] },
    { "label": "젖소",     "series": [ {"year":2023,"value":387} ] },
    { "label": "돼지",     "series": [ {"year":2023,"value":11077} ] },
    { "label": "닭",       "unit": "천수", "series": [ {"year":2023,"value":178000} ] }
  ]
}
```

## 5. 세 기능이 이 스키마를 쓰는 방법

- **① 대시보드**: `is_headline=true`인 12개를 첫 화면에 카드+꺾은선. 그룹 탭으로 나머지 접근. `series`/`breakdown`만 읽으면 됨.
- **② 검색**: 1차는 `name`·`keywords`·`group`로 구조화 필터. 2차(RAG)는 `description`+원문 텍스트를 임베딩해 자연어 질의.
- **③ AI 분석·조언**: 질의에 맞는 레코드들을 골라 **실제 `series` 수치만** 근거로 추론(기억으로 숫자 생성 금지). `related_ids`로 지표를 자동으로 엮고, `source`로 출처를 인용.

- **시계열**: **전체(1970~2024)**를 데이터로 보관. 대시보드 기본 표시는 초안에서 눈으로 보고 조정.
