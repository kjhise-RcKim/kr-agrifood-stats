# 프로젝트 인수인계 (README) — 농식품 주요통계 대시보드

> 이 문서는 웹 Claude와 논의해 확정한 내용을 Cowork로 넘기기 위한 요약본입니다.
> Cowork Claude는 이 파일과 `핵심지표_구조_초안.md`를 먼저 읽고 작업을 이어가면 됩니다.

## 1. 무엇을 만드나
농림축산식품부 『농림축산식품 주요통계 2025』(581p PDF)를 일반 공개용 웹 도구로 만든다.
원문이 방대하고 사전처럼 나열돼 있어 활용성이 낮은 문제를 해결하는 것이 목표.

## 2. 제품 = 3개 층
1. **대시보드** — 핵심지표를 카드+그래프로 한눈에
2. **검색** — 자연어로 원하는 자료 찾기 (구조화 필터 → 이후 RAG)
3. **AI 분석·조언** — 지표를 엮어 인사이트·판단·조언 (반드시 실제 수치 근거, 숫자 생성 금지)

## 3. 확정된 결정
- 지표 **30개 전부 + 외국·남북한 비교 통계 포함** (축소 없이 원문에 충실)
- 시계열은 **전체(1970~2024)** 보관, 대시보드 기본 표시 범위는 초안 보고 조정
- **핵심 설계 원칙**: 대시보드·검색·AI가 **같은 JSON 하나**를 읽는다 (지표 1개 = 레코드 1개)
- 모든 레코드에 `source`(장·페이지) 필수 → 공공누리 출처표시 + AI 인용 근거
- 상세 스키마와 지표 목록은 `핵심지표_구조_초안.md` 참조

## 4. 만드는 순서
데이터 정리 → ①대시보드 → ②검색 → ③AI 분석·조언
(앞 단계 결과물이 뒤 단계 재료. AI는 맨 마지막.)

## 5. 운영·기술 결정
- **GitHub 먼저 연결**: 백업 + 되돌리기(바이브코딩 필수) + 회사/집 이어작업 + 무료 공개
- 공개 배포: GitHub Pages 또는 Vercel (무료)
- 폴더는 이 프로젝트 **전용 폴더 하나만** (민감파일 노출 방지)
- ③ AI 기능은 API 키를 프론트에 노출하면 안 됨 → 키 숨기는 가벼운 백엔드(서버 함수) 필요. ①②까지는 프론트만으로 가능.

## 6. 이 폴더에 있어야 할 파일
- `프로젝트_인수인계.md` (이 문서)
- `핵심지표_구조_초안.md` (지표 30개 + 비교통계 + JSON 스키마)
- `2025_농림축산식품_주요통계.pdf` (원자료 — 데이터 추출용)

## 7. Cowork가 이어서 할 일
1. 위 파일들을 프로젝트 폴더의 `/docs`에 정리 (스키마는 `/docs/schema.md`)
2. 이 폴더를 Git 저장소로 만들고 GitHub에 연결
3. PDF에서 30개(+비교) 지표에 해당하는 표만 뽑아 스키마대로 JSON 변환 (581p 전부가 아니라 해당 표만)
4. ①대시보드 v1 착수 → 초안 확인 후 헤드라인 12개·시계열 기본표시 조정
