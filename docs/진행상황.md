## B. ⓪ 지표 딥링크 구현 완료 ✅ (쿼리 방식)
`index.html`만 수정(백엔드 무관).

| 주소 | 화면 |
|---|---|
| `/` | 홈 |
| `?i=<지표id>` | 지표 상세 (예: `?i=food_crop_production`) |
| `?g=<그룹명>` | 그룹 탭 |
| `?q=<검색어>` | 검색 결과 |
| `?view=compare` | 즐겨찾기 겹쳐보기 |
| `&t=dark` / `&t=light` | **테마(배경)** — 모든 링크에 자동으로 붙음 |

**테마 동행**: 공유자가 보던 밝게/어둡게 모드가 링크에 실려 상대방도 같은 배경으로 열린다.
`urlFor()`가 모든 주소에 `t=`를 자동 부착하고, `applyURL()`이 렌더 **전에** `applyTheme()`을 호출(그래프 색까지 반영).
테마 토글 시 `replaceURL(baseQS())`로 주소의 `t=`도 갱신 → 주소창을 그대로 복사해도 맞다. `baseQS()`는 현재 주소에서 `t`만 제거한 나머지.

> ### ⚠️ 왜 해시(#)가 아니라 쿼리(?)인가 — 중요
> 처음에 `#/i/…` 해시 라우터로 만들었더니 **카카오톡 등 SNS 링크 미리보기(OG)가 사라졌다.**
> `#` 뒤쪽은 **서버로 전송되지 않아** 미리보기 수집기가 새 주소로 인식하거나 아예 미리보기 생성을 건너뛴다.
> → **쿼리(`?`)로 전환.** 쿼리는 서버까지 전달되는 정식 주소라 OG가 정상 동작한다. 앞으로도 해시 라우팅으로 되돌리지 말 것.
> (OG 태그 자체는 이상 없었음: `og:image`·`og:url` 모두 절대경로로 정상.)
> 향후: 쿼리 방식이라 **지표별 OG 이미지**(서버리스 함수로 `?i=`에 따라 다른 미리보기)도 확장 가능.

- `migrateHash()` — 이미 공유된 예전 `#/i/…` 주소를 접속 시 `?i=…`로 자동 전환(하위 호환).
- `history.pushState`/`popstate` 사용(새로고침 없음). 검색은 `replaceState`라 히스토리 미적재.

- 새 함수: `go()` `applyURL()` `replaceURL()` `urlFor()` `migrateHash()` `visibleIn()` `findInd()` `shareURL()` `copyLink()` + `popstate` 리스너
- 카드·탭·겹쳐보기 클릭 → 직접 렌더 대신 **`go()`로 주소 변경 → `applyURL()`가 렌더**(단일 경로). 초기 진입은 `migrateHash(); applyURL();`
- 상세 화면에 **`🔗 링크 복사`** 버튼 추가(`dl-link`). `navigator.clipboard` → 실패 시 `textarea+execCommand` 폴백, 버튼 라벨로 결과 표시.
- `findInd()`는 **id뿐 아니라 지표명(공백 무시)로도** 조회 가능 → `#/i/식량작물생산량` 형태도 동작.
- `visibleIn()`으로 **현재 탭에 그 지표가 보이면 탭을 바꾸지 않음** (전체 탭에서 헤드라인 카드 눌렀을 때 탭이 튀는 것 방지).
- 알 수 없는 해시는 조용히 홈으로.
- 이용안내에 **「🔗 링크로 콕 집어 공유하기」** 항목 추가.
- 검증: 스크립트 추출 후 `node --check` 통과, data.js 34지표 로드 확인.

5. **역대 최고/최저 경신**(유효값 6개 이상, max≠min)이면 점수 ×1.6
6. 표시: `%`·`%p` 단위 지표는 **%p**, 나머지는 **%** (사용자 지적 반영 — 밀 자급률 0.6→0.7이 +17%로 1등 먹는 문제 방지)

### G-2. 브리프 확장 구조로 전환 ✅ (같은 날)
**문제 제기(사용자)**: 브리프가 늘어나면 헤더 버튼이 계속 늘어나 공간이 없어진다.
**해결**: 헤더 버튼은 **`📑 브리프` 하나로 고정**하고 브리프는 안에서 늘린다.

- **`web/briefs.js` 신설** — 브리프 정의를 **데이터로 분리**. 새 브리프는 이 배열에 항목 하나만 추가하면 되고 **코드 수정 불필요**.
  - `src` 형식: `[지표id]` = 계 시계열 / `[지표id,'품목명']` = breakdown 시계열
  - `axis`: `'y'`(왼쪽) / `'y2'`(오른쪽), 단위 같으면 둘 다 `'y'`
  - `color:'__TOTAL__'`는 테마색(`--total`)으로 치환
  - 파일 상단에 **"해석·주장 문장 금지" 원칙**을 주석으로 박아둠
- **화면 3종**
  - `?brief=list` — 브리프 목록(카드). 각 카드에 대표 KPI 최신값 표시
  - `?brief=<id>` — 개별 브리프. 상단에 **이동 칩**(좌우 스크롤) + `📑 전체 목록`
  - 첫 화면(대표지표)에 **`📑 주제별로 모아보기` 바로가기 줄**(`#bfhome`) — 발견성 확보
- **`📑 브리프` 버튼**은 브리프가 1개면 바로 그 브리프로, 2개 이상이면 목록으로.
- 알 수 없는 `?brief=` 값은 조용히 일반 화면으로.
- 기존 `renderWheat()` 삭제 → **범용 `renderBriefPage(B)` / `renderBriefList()`** 로 대체. 차트는 `briefCharts[]`에 모아 재렌더 시 destroy.
- 테마 변경 시 `curBrief`가 있으면 해당 브리프 재렌더.

- **별도 랜딩 페이지는 반대** → 채택 안 함. 이유: ⑴ `?i=`·`?brief=` 딥링크를 막아 공유 효과를 죽임 ⑵ 재방문자에게 벽 ⑶ 화려한 모션은 '통계 도구'가 아닌 '홍보 사이트'로 읽힘.
- **홈 히어로 섹션**으로 결정(사용자 선택). 모션은 **절제된 수준**.
- 문구는 사용자가 ①질문형 제목 + ③담백형 부제를 조합해 확정:

  > 『농림축산식품 주요통계 2025』를 그래프·검색·AI로.
  > 숫자는 원자료 그대로, 근거 쪽수와 함께.
- ⚠️ **"한국은 먹을 것의 절반을 수입합니다" 류의 주장 문구는 쓰지 않는다.** 히어로는 *도구가 무엇인지*만 말하고, 숫자도 *도구에 대한 숫자*(지표 46 / 브리프 12 / 원문 581쪽)만 쓴다. 브리프의 "해석·주장 없음" 원칙과 동일.

- **표시 조건**: `homeView && !query && !compareMode && curGroup==='전체'`.
  `homeView`는 **`applyURL()`의 홈 분기에서만 true** → `?i=`·`?brief=`·`?q=`·`?view=`로 들어오면 히어로 없이 바로 목적 화면. `setBriefPage(true)`도 `hero`를 숨김.
